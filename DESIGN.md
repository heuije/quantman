# 디자인 시스템 — 퀀트플랫폼

웹앱과 로컬앱의 UI는 이 문서를 기준으로 한다. 새 화면·컴포넌트를 만들 때
색상·간격·패턴을 여기서 가져다 쓰고, 벗어날 때는 먼저 합의한다.

## 제품 맥락

- **무엇:** 한국 주식 자동매매 플랫폼. 웹앱에서 매매 전략을 수립하고,
  사용자 PC의 로컬앱이 그 전략으로 모의투자를 자동 실행한다.
- **대상:** 퀀트 트레이딩을 시작하려는 초중급 개인투자자.
- **표면:** 웹앱(React SPA, 전략 수립) + 로컬앱(Python Tkinter, 자동 실행).

## 디자인 원칙

1. **흐름을 보이게.** 전략 만들기 → 모의투자 배정 → 기기 연결 → 자동매매 가동.
   사용자는 늘 "지금 어디에 있고 다음에 뭘 할지" 알 수 있어야 한다.
2. **숫자를 해석해 준다.** 지표를 그대로 던지지 않는다. 등급·평이한 문장·
   툴팁으로 초중급 사용자가 판단할 수 있게 한다. (제품의 핵심 차별점)
3. **신뢰감.** 자동매매에 돈을 맡기는 도구다. 회색 기본 위젯이 아니라
   일관된 톤, 명확한 상태 표시, 빈 상태 안내로 신뢰를 준다.

## 아이덴티티

- **방향:** 클린 SaaS에 한국 트레이더용 정체성을 입힌다. Claude 계열 따뜻한
  테라코타(주황) 브랜드 액센트 + 잉크(따뜻한 차콜) 주요 버튼 + 크림 배경 + 카드형 패널.
- **장식 수준:** 미니멀. 타이포·여백·테두리가 일한다. 그라데이션·장식 없음.
- **차별 포인트:** ① Pretendard로 글자에 얼굴을 준다. ② 손익 색은 **한국 관례
  (상승·수익=빨강 / 하락·손실=파랑)** — 연동 브로커(KIS)·국내 증권사 화면과 일치.
  ③ 브랜드 강조는 따뜻한 테라코타(`#d97757`)지만 **UI 크롬에만** 쓰고, 수익 빨강은
  **숫자에만** 써서 같은 난색이라도 맥락으로 분리. 주요 버튼은 잉크 차콜.
  ④ 중성색을 살짝 따뜻하게(크림) 해 주황이 겉돌지 않고 native하게 어울린다.

## 색상

웹앱 토큰은 `web/src/index.css`의 `:root`, 로컬앱 상수는
`local/localapp/gui.py` 상단에 동일 값으로 정의되어 있다.

**브랜드/표면**

| 역할 | HEX | 용도 |
|------|-----|------|
| bg | `#faf9f6` | 페이지 배경 (따뜻한 크림) |
| panel | `#ffffff` | 패널·카드 |
| border | `#e8e3db` | 테두리·구분선 (따뜻한 그레이) |
| text | `#20201d` | 본문 (따뜻한 근검정) |
| muted | `#6f6a62` | 보조 텍스트 (따뜻한 그레이) |
| accent | `#d97757` | **테라코타(주황)** — 활성 nav·링크·포커스·문장빌더 강조 |
| accent-soft | `#f7ece5` | 주황 배경 틴트 (따뜻한 피치) |
| ink / ink-hover | `#292524` / `#423c37` | **주요 버튼** 채움 (따뜻한 차콜) |

**시장 방향 (한국식 — 손익·가격 방향 전용)**

| 역할 | HEX | 용도 |
|------|-----|------|
| up / up-soft | `#e5383b` / `#fdeceb` | 상승·수익·매수 |
| down / down-soft | `#1668c4` / `#e7f0fa` | 하락·손실·매도 |

**상태/품질/위험 (시장 방향과 별개)**

| 역할 | HEX | 용도 |
|------|-----|------|
| green / green-soft | `#15803d` / `#ecfdf3` | 정상·완료·연결됨 |
| red / red-soft (=danger) | `#b91c1c` / `#fef2f2` | 오류·위험·**실전모드 경고** |
| amber / amber-soft | `#b45309` / `#fffbeb` | 주의·설정 미완료 |

의미 규칙:
- **방향성 숫자**(수익률·손익·가격 변화·매수/매도) = up(빨강) / down(파랑). `.pos`/`.neg`.
- **상태/품질** = green(정상·완료·좋은 등급) / red(오류·위험·실전경고) / amber(주의).
- **주요 액션** = ink(버튼 채움), **브랜드 강조·활성** = accent(틸).
- 주의: 수익=빨강(`--up`)과 위험·실전경고=빨강(`--red`)이 둘 다 빨강 계열이나,
  전자는 밝은 빨강 숫자색·후자는 진한 빨강 구조 프레임/뱃지로 맥락이 다르다.

## 타이포그래피

- **스택:** `"Pretendard Variable", Pretendard, -apple-system, "Segoe UI",
  "Malgun Gothic", Roboto, sans-serif`. Pretendard(가변폰트, jsDelivr CDN)가
  한글·영문을 함께 처리해 글자에 일관된 얼굴을 준다. 로드 실패 시 OS 폰트로 폴백.
- **숫자:** `:root`에 `font-variant-numeric: tabular-nums` — 표·KPI·수익률의
  숫자가 고정폭으로 세로 정렬된다. 트레이딩 화면 신뢰감의 핵심.
- **데이터/로그:** 등폭(`Consolas`) — 로컬앱 로그 영역.
- **크기:** 페이지 제목 22 / 패널 제목 15 / 문장빌더·본문 14~15 / 보조 12 / 캡션 11.

## 간격·레이아웃

- 패널 `padding 20px`, 패널 간 `margin-bottom 18px`.
- 모서리 반경 `--radius: 10px` (작은 칩·버튼은 7~8px).
- 웹앱 본문 최대폭 1100px, 좌측 220px 사이드바.
- 로컬앱 창 560×740, 섹션 간 `padx 12 / pady 4~6`.

## 컴포넌트 패턴

- **패널(`.panel`):** 흰 배경 + 1px 테두리 카드. 모든 콘텐츠 묶음의 기본 단위.
- **지표 카드(`.stat`):** 라벨 + 큰 값. 값에 `.pos`/`.neg`로 손익 색.
- **지표 툴팁(`.metric-hint`):** 라벨 옆 `?` 호버 시 한 줄 설명. 전문용어
  (MDD·샤프·CAGR·p-value 등)에는 반드시 붙인다.
- **결과 해석 카드(`Verdict`):** 백테스트 결과 상단. 등급 뱃지 + 한 줄 요약 +
  항목별 코멘트(pt-good/pt-warn/pt-bad). tone(good/ok/warn/bad)로 좌측
  보더·배경 틴트 결정.
- **진행 단계(`.steps`/`.step`):** done(✓·green) / current(번호·accent 강조,
  CTA 노출) / todo(회색). 온보딩 여정을 항상 보이게 한다.
- **하단 액션바(`.action-bar`):** `position: sticky; bottom`. 주요 액션을
  스크롤과 무관하게 늘 닿게 한다. 기본 액션 1개 + 보조 액션.
- **빈 상태(`.empty-state`):** 회색 한 줄 금지. 제목 + 안내 문단 + CTA 버튼 카드.
- **문장형 빌더(`.sentence`/`.chip`):** 제품의 핵심 차별점 → 시각적 주인공.
  각 조건 줄은 흰 패널 + **테라코타 좌측 보더 3px**로 강조. 칩은 채워 넣는 토큰처럼
  (`.chip` = 주황 틴트 배경, 호버 시 주황 보더), 빈 칩(`+수식어`)은 점선·투명 배경으로
  "비었음"을 구분. 연결어("가/이"·"일 때")는 muted 15px로 빈칸 채우기 리듬을 살린다.
  `[이 종목]` placeholder 강조는 테라코타(`--accent`)로.
- **버튼:** 기본 = **ink 채움**(검정 계열, 시장 빨강·파랑과 비충돌). 보조 =
  `.ghost`(연회색). 작은 건 `.sm`. 위험 액션은 `.danger-btn`(red).
- **로컬앱:** ttk `clam` 테마 + 위 팔레트. 상단 상태 히어로(앰버=미완료 /
  그린=실행 중 / 슬레이트=중지), 섹션은 ①②③ 번호 + 헤더에 ✓ 진행 표시.

## 모션

미니멀-기능적. 색·투명도 전환만(120ms 내외). 등장 애니메이션·스크롤 연출 없음.

## 자동매매 정책 (Phase 9A부터)

베타 단계의 자동매매는 다음 best practice를 글로벌 default로 적용한다. 전략별
override는 `Strategy.execution` 필드로 가능. 자세한 근거는 별도 리서치 노트 참조.

1. **시장가 → 지정가 + price tolerance.** 신규 진입은 어제 종가 × (1 + 1%)
   지정가, 청산은 −2% 지정가. 미체결 시 5분 후 자동 취소 → 그날 신호 폐기.
   호가 단위 자동 라운딩.
2. **갭 필터.** 전일 종가 대비 현재가 갭이 ±2.5% 초과 시 신규 진입 폐기.
   매도 시그널은 손절을 위해 갭 필터 무시 (단 limit tolerance로 보호).
3. **일일 손실 한도 + kill switch.** 자본 대비 −3% 도달 시 자동 전량 청산 +
   신규 차단. 사용자가 명시적 reset 전까지 유지.
4. **ATR 변동성 보정 사이징.** 수량 = (자본 × 1% risk) ÷ (ATR × 2). 단일
   종목 비중은 자본의 10%로 클램프. 레거시 `amount_pct` 모드(`sizing_mode:
   pct_cash`)도 지원.
5. **백테스트 슬리피지·갭 가정.** 편도 25bps 수수료 + 10bps 슬리피지 default,
   갭 1% 초과분의 절반을 추가 비용으로 산입. 라이브에서는 의도가 vs 체결가를
   bps로 누적 측정해 백테스트 가정과 비교 가능하게 함.
